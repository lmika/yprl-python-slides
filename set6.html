Step 1: Dictionaries

Values That Name Values

Types So Far

  • Strings: Sequence of characters
  • Integers: Numbers without fractional parts
  • Floats: Numbers with fractional parts
  • Booleans: Either True or False
  • Lists: An order sequence of values

Look at one more today, the Dictionary.

Real Life Dictionaries

  • A collection of words
  • Each word has an associated definition
  • You can get a definition by it's associated word in any order you like

Python Dictionaries

words = {
    "diction": "manner of citation in...",
    "dictionary": "a book that lists this words...",
}
  • A collection of values (called names)
  • Each name has an associated value (called the value)
  • These make up a collection of name/value pairs
  • You can get a value by it's associated name in any order you like

Dictionaries

Holds a collection of names and associated values:

person = { "name": "Harry", "age": 213, "gender": "M" }

Dictionaries

To get the value for a name, use [] (just like lists):

person["name"]      # Returns "Harry"

Dictionaries

To change the value of a name, or to add a new name, use [] with =:

person["name"] = "Jeff"
person["address"] = "123 Fake St."

Dictionary Length

To return the number of name/value pairs, use the len keyword:

len(person)         # 3

Looking Up A Name

To see if a name is in a dictionary, use the in operator.

"age" in person           # True
"suburb" not in person    # True

in and not in works for lists as well.

Dictionaries And Lists

Lists

  • Values are ordered
  • Addressable by an integer index that starts at 0
  • Hold a collection of related values

Dictionaries

  • Values are not ordered
  • Addressable by a name which can be a number, string or boolean
  • Hold a set of associated name, value pairs

Step 2: Defining Functions

Thinking At A Higher Level

Functions Revistied

We've seen functions like input, int and len a few times now:

value = int(input("Enter a number: "))

Each one does a particular operation for us which saves us from writing the code to do it ourselves.

We can do this using our own functions.

Properties Of A Function

  • Each one has a name (e.g. input)
  • Each one has an associated block of code, called the body
  • Each one can have zero or more arguments or input parameters* (which are stored as variables)
  • Each one can return a result

When Using A Function

To use (or call) a function, the following steps happen:

  • Take the input values
  • Look for the function body
  • Assign them to variables used by the function
  • Run the function body
  • If the body has a value to return, pass that value back to the caller

Defining Our Own Functions

To define a function, start by typing def, followed by the function name:

def add_2_numbers(...):
    ...

Defining Our Own Functions

If you want your function to take arguments, type them as variable names within the brackets:

def add_2_numbers(left, right):
    ...

When doing this, the function caller must suply values for each argument:

good = add_2_numbers(1, 1)          # Works
bad = add_2_numbers(1)              # Error

Arguments are local to the function (i.e. they cannot be used outside the function)

Defining Our Own Functions

Then, type the code you want the function to do:

def add_2_numbers(left, right):
    addition = left + right

Any new variables like addition are also local to the function.

If we don't want to return a value, we are done.

To return a value, we have one more thing to do:

Defining Our Own Functions

To return a value to the caller, use the return keyword:

def add_2_numbers(left, right):
    addition = left + right
    return addition

Now, we can use this function to add two numbers togeather:

val = add_2_numbers(2, 3)           # Store 5 in val

As programs get larger, it becomes necessary to reason about them at a higher level:

Step 3: Tidbits

A Few Other Things

The elif Statement

Sometimes if statements may have more than one test:

if (x < 5):
    print("x is less than 5")
else:
    if (x > 5):
        print("x is greater than 5")
    else:
        print("x is 5")

However, having too many else/ifs will push the code to the right.

The elif Statement

This makes it easier to have multiple choices within one if statement:

if (x < 5):
    print("x is less than 5")
elif (x > 5):
    print("x is greater than 5")
else:
    print("x is 5")

Accessing Globals From A Function

Each function has their own set of variables called locals.

addition = 1 + 1

def add_2():
    addition = addition + 2         # Different "addition",
    return addition                 # will cause an error

def add_4_numbers(a, b, c, d):
    addition = a + b + c + d        # Different "addition"
    return addition                 # again

These locals override (or shadow) globals that are outside the function.

The global Keyword

In order to use a global, you need to declare the variable as being global:

addition = 1 + 1

def add_2():
    global addition
    addition = addition + 2
    return addition

add_2()     # Returns 4, addition = 4
add_2()     # Returns 6, addition = 6
